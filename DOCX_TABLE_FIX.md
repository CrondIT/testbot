# Исправление проблемы с отображением таблиц в DOCX

## Проблема

Функция `DocxRenderer` в модуле `docx_utils.py` не создавала таблицы в DOCX-файле, несмотря на то, что метод `_table` был реализован корректно.

## Причина

Основной причиной проблемы было то, что инструкция для модели ИИ в `handle_utils.py` не включала пример структуры JSON для таблиц. В инструкции, которая добавляется при запросе формата Word, были указаны только примеры для заголовков и параграфов:

```json
{
  "meta": {"title": "string"},
  "blocks": [
    {"type":"heading","level":1,"text":"string"},
    {"type":"paragraph","text":"string"}
  ]
}
```

Но не было примера для таблиц:
```json
{"type":"table", "headers":["column1", "column2"], "rows":[["value1", "value2"]]}
```

Поэтому модель ИИ не знала, что нужно генерировать JSON с блоками типа "table".

## Решение

Было обновлено содержание инструкции для модели ИИ в `handle_utils.py`, добавлены примеры для всех поддерживаемых типов блоков, включая таблицы:

```json
{
  "meta": {"title": "string"},
  "blocks": [
    {"type":"heading","level":1,"text":"string"},
    {"type":"paragraph","text":"string"},
    {"type":"list", "ordered":false, "items":["item1", "item2"]},
    {"type":"table", "headers":["column1", "column2"], "rows":[["value1", "value2"], ["value3", "value4"]]}
  ]
}
```

## Результат

После этого изменения модель ИИ стала генерировать JSON, который включает блоки с типом "table", и `DocxRenderer` успешно создает таблицы в DOCX-файле.

## Тестирование

Были созданы тесты, которые подтверждают, что:

1. `DocxRenderer` корректно обрабатывает JSON с табличными данными
2. Метод `_table` работает правильно при наличии корректных данных
3. Крайние случаи (пустые таблицы) обрабатываются надлежащим образом
4. Все типы блоков (заголовки, параграфы, списки, таблицы) работают вместе
